# Database Migrations with Alembic

This directory contains database migration scripts managed by Alembic. Migrations allow you to evolve your database schema over time in a consistent and reproducible way.

## Prerequisites

Ensure you have the backend dependencies installed and your virtual environment activated:

```bash
# From the backend directory
pip install -r requirements.txt
```

Ensure your `.env` file is configured with the correct `DB_*` variables, as the migration script uses these to connect to your database.

## Common Commands

All commands should be run from the `backend/` directory (where `alembic.ini` is located).

### 1. Apply Migrations
Update your database to the latest schema:
```bash
alembic upgrade head
```

### 2. Create a New Migration (Manual)
Create an empty migration script:
```bash
alembic revision -m "description_of_change"
```

### 3. Create a New Migration (Autogenerate)
Alembic can compare your current SQLAlchemy models with the database schema and generate the migration automatically:
```bash
alembic revision --autogenerate -m "description_of_change"
```
*Note: Always review autogenerated scripts in `migrations/versions/` before applying them.*

### 4. Rollback
Revert the last migration:
```bash
alembic downgrade -1
```
Revert to a specific revision:
```bash
alembic downgrade <revision_id>
```

### 5. View Status
Show current revision:
```bash
alembic current
```
Show migration history:
```bash
alembic history --verbose
```

## Typical Workflow

1.  **Modify Models**: Update your SQLAlchemy models in `app/models/article.py`.
2.  **Generate Migration**: Run `alembic revision --autogenerate -m "Your description"`.
3.  **Review**: Open the newly created file in `migrations/versions/` to ensure it captures your changes correctly.
4.  **Apply**: Run `alembic upgrade head`.

## Important Notes

- **Naming**: Use descriptive messages for your revisions.
- **Committing**: Always commit your migration scripts (`migrations/versions/*.py`) to version control.
- **Production**: Never run `autogenerate` directly against a production database without manual review first.
